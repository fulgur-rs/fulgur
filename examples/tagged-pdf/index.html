Quarterly Accessibility Report

This document demonstrates fulgur's Tagged PDF output. When rendered with --tagged or --pdf-ua, the PDF contains a structure tree that screen readers and accessibility tools can traverse.

Summary

Tagged PDF embeds semantic structure — headings, paragraphs, lists, tables, figures, and links — directly in the PDF file. This structure is required for PDF/UA-1 conformance and enables assistive technologies to present documents in a meaningful order.

Supported Elements

The following HTML elements map to PDF structure tags:

HTML element PDF structure tag
h1–h6H1–H6
p, divP, Div
spanSpan
ulL (unordered)
olL (ordered)
liLI → Lbl + LBody
tableTable
thead / tbody / tfootTHead / TBody / TFoot
trTR
thTH (scope: Row/Column/Both)
tdTD
imgFigure (alt text propagated)
aLink

Step-by-Step Workflow

  1. Author semantic HTML using the elements listed above.
  2. Run fulgur with --tagged (structure tree) or --pdf-ua (full PDF/UA-1).
  3. Verify the output with a PDF accessibility checker such as PAC 2024 or Adobe Acrobat's accessibility tool.

Known Limitations

  • Alt text is propagated from img alt attributes only; CSS background images are not tagged.
  • PDF/UA-1 requires a document language. Always pass --lang (CLI) or .lang() (API) when using --pdf-ua.
  • Python and Ruby bindings do not yet expose tagged / pdf_ua options.

CLI Usage

Generate a Tagged PDF:

fulgur render --tagged --language en -o report.pdf report.html

Generate a PDF/UA-1 conformant PDF (implies --tagged and --bookmarks):

fulgur render --pdf-ua --language en -o report.pdf report.html
